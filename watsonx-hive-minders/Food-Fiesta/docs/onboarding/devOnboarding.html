1 — Purpose

Food Fiesta is a Spring Boot fullstack dining management platform. It allows customers to browse a food menu, register accounts, log in, and place orders. Restaurant administrators manage the full catalogue of products, users, admins, and orders from a protected back-office dashboard.

Main User Roles

RoleAccessEntry point
AdminFull CRUD on users, admins, products, orders. Back-office dashboard./login → /admin/services
CustomerBrowse menu, register, log in, search products, place and view orders./login → /dashboard
Google OAuth2Same as Customer — auto-provisioned on first Google login./login → Google → /oauth2/success

Core Features

  • Thymeleaf server-side rendered UI with modern CSS/JS
  • Dual login flow: custom form-based (admin & customer) + Google OAuth2
  • Product inventory management (add, edit, delete food items)
  • User self-registration and admin-managed user accounts
  • Order placement with automatic total calculation and order history
  • Spring Data JPA persistence (H2 in-memory for local dev, PostgreSQL for Docker/prod)
  • Swagger/OpenAPI documentation at /swagger-ui/index.html

Authentication Mechanism

Authentication is custom session-based, not Spring Security's built-in UserDetailsService. Spring Security is configured only to protect routes and enable OAuth2 — the actual credential validation is done manually in service methods.

  • Admin login — POST /adminLogin → AdminServices.validateAdminCredentials() compares plain-text password directly against the DB value. On success, stores the admin email in HttpSession as "loggedInAdmin".
  • Customer login — POST /userLogin → UserServices.validateLoginCredentials() iterates all users and compares plain-text passwords. On success, stores the full User JPA entity in HttpSession as "loggedInUser".
  • Google OAuth2 — handled by Spring Security. On success, redirects to /oauth2/success where OAuth2Controller auto-provisions a local User record (password set to placeholder "OAUTH_USER") and stores it in session.
  • Public routes: /, /home, /login, /register, /adminLogin, /userLogin, /products, /location, /about, plus all static assets and Swagger/H2 console.
  • CSRF is disabled. Passwords for users and admins are stored in plain text (no BCrypt applied, despite PasswordEncoder bean being declared).

7 — Known Gotchas & Contribution Guide

Known Gotchas

Plain-text passwords — do not "fix" the login methods without a migration plan. Both AdminServices.validateAdminCredentials() and UserServices.validateLoginCredentials() compare passwords in plain text. A BCryptPasswordEncoder bean is declared in SecurityConfig but is never injected or used. If you encode passwords in one place, you must migrate all stored passwords — coordinate with the team before touching this.
Full JPA entity stored in HttpSession. session.setAttribute("loggedInUser", loggedInUser) stores the entire User JPA entity (including its lazy orders collection) in the HTTP session. This can cause LazyInitializationException after session serialisation. Always re-fetch the user from the DB or store only the ID.
Optional.get() called without isPresent() guard. AdminServices.getAdmin(), UserServices.getUser(), ProductServices.getProduct(), and ProductServices.updateproduct() all call optional.get() directly. Passing a non-existent ID will throw NoSuchElementException at runtime. Do not remove the guard-less calls without also adding error handling.
Services annotated @Component instead of @Service. AdminServices, UserServices, ProductServices, and OrderServices are all annotated @Component. This works (Spring treats them identically at runtime) but is intentional styling — do not "correct" it to @Service without team discussion.
Mutation endpoints mapped to @GetMapping. /updatingAdmin/{id}, /updatingProduct/{productId}, /updatingUser/{id}, /deleteAdmin/{id}, /deleteProduct/{productId}, and /deleteUser/{id} are all @GetMapping. This is by design (HTML forms can only GET/POST). Do not change verb without also updating the Thymeleaf templates.
OrderController is an empty stub. The class exists with imports but contains no methods. Do not assume any order-related routing flows through it — all active order handling is in AdminController.
Field name totalAmmout has a deliberate typo. The Orders entity field is totalAmmout (double-m). This maps to a DB column of the same name. Never rename it without a DB migration.
Default admin password is hardcoded in DataLoader. The seed admin account (admin@foodfiesta.com / admin123) is created unconditionally if the admin table is empty. For any non-local environment, change this before the first startup.
CSRF protection is disabled. SecurityConfig calls .csrf(csrf -> csrf.disable()). This is necessary because the app uses plain HTML forms without CSRF tokens, but it is a known security gap. Any future API endpoints should use token-based protection.
UserServices.validateLoginCredentials() is O(n). It loads all users from the DB and iterates them to find a matching email+password. Replace with a query (findByUemailAndUpassword) if the user count grows.

Contribution Guide

Run tests locally

./mvnw test       # macOS / Linux
.\mvnw.cmd test   # Windows

All existing tests must pass before pushing. The single test class is FoodFrenzyApplicationTests (Spring context load test).

Run the app locally before pushing

./mvnw spring-boot:run

Branching convention

The CI pipeline triggers on all branches and all pull requests. Use feature branches and merge to main via pull request. Direct pushes to main are not blocked by the pipeline but should be avoided.

CI checks on push

GitHub Actions runs automatically on every push: checkout → Java 21 setup → ./mvnw test. The build must be green before merging.

Adding new service methods

If you add a new service method, add a matching test in the corresponding *Test.java file. Currently the only test file is FoodFrenzyApplicationTests.java — you will need to create dedicated unit test files as the suite grows.

Adding a new endpoint

  1. Add the handler method to the appropriate controller (or create a new @Controller class).
  2. If the endpoint requires authentication, add a session guard (if (session.getAttribute("loggedInUser") == null) return "redirect:/userLogin";).
  3. Add the route to SecurityConfig.authorizeHttpRequests() if it should be public.
  4. Add a Thymeleaf template under src/main/resources/templates/ if needed.
  5. Document the endpoint with @Operation and @Tag annotations for Swagger.
