|---|---|---|
| 🌐 Client | Thymeleaf SSR, HTML Forms (POST/GET), CSS + JavaScript (static) | Server-side rendered — no SPA framework |
| 🔒 Spring Security (custom session auth) | Route whitelist (permitAll), Google OAuth2 login, HttpSession (loggedInUser / loggedInAdmin) | CSRF disabled ⚠️; BCryptPasswordEncoder declared but **never used** ⚠️; passwords stored in plain text ⚠️ |
| 🎮 Controllers (@Controller) | HomeController, AdminController ⚠️, UserController, ProductController, OAuth2Controller, OrderController ⚠️ | AdminController is a God class (auth + order + admin CRUD). OrderController is an **empty stub** ⚠️ |
| ⚙️ Services (annotated @Component, not @Service) | AdminServices ⚠️, UserServices ⚠️, ProductServices ⚠️, OrderServices | All use `Optional.get()` without guard ⚠️; UserServices.validateLoginCredentials() is O(n) ⚠️ |
| 🗄️ Repositories (Spring Data JPA) | AdminRepository (CrudRepository), UserRepository (CrudRepository), ProductRepository (CrudRepository), OrderRepository (JpaRepository) | Custom queries: findByAdminEmail, findUserByUemail, findByPnameContainingIgnoreCase, findOrdersByUser |
| 💾 Data Stores | customer_info ── User, admin ── Admin, product_table ── Product, order_info ── Orders | H2 in-memory (local); PostgreSQL (Docker / production) |
| 💡 HTTP Session | loggedInAdmin → String (email), loggedInUser → User (full JPA entity) | Storing a full JPA entity in session can cause LazyInitializationException ⚠️ |

`Browser` → `Spring Security filter chain` → `Controller (session guard)` → `Service (business logic)` → `Repository (JPA)` → `H2 / PostgreSQL` → `Thymeleaf template` → `HTML response`
