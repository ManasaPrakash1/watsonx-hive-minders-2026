| ⚙️ Services (annotated @Component — not @Service) | AdminServices ⚠️ plaintext password comparison, UserServices ⚠️ plaintext password + full-table scan login, ProductServices, OrderServices, Logic (static utility) | All services use @Component instead of @Service — intentional; Optional.get() without guard in all services |
| 🗄️ Repositories | AdminRepository, UserRepository, ProductRepository, OrderRepository | Spring Data JPA — extends JpaRepository / CrudRepository |
| 💾 Database | H2 in-memory (local dev — create-drop), PostgreSQL (Docker / prod), customer_info ── User, admin ── Admin, product_table ── Product, order_info ── Orders (FK user_u_id) | Dev: H2 resets on every restart. Prod: PostgreSQL via Docker or cloud. |
| 💡 HTTP Session | loggedInUser → User entity (full JPA object) ⚠️, loggedInAdmin → String (admin email) | ⚠️ Storing a full JPA entity in session can cause LazyInitializationException |

## Flux de requête

`Browser` → `Spring Security FilterChain` → `Controller` → `Service (@Component)` → `Repository (JPA)` → `H2 or PostgreSQL` → `Thymeleaf Template` → `HTTP Response`
