🚨 Alertes proactives

🔴 CRITIQUE — Mots de passe en clair
Les mots de passe des administrateurs et des clients sont comparés en texte clair dans AdminServices.validateAdminCredentials() et UserServices.validateLoginCredentials(). En cas de fuite de la base de données (injection SQL, backup exposé, accès non autorisé), tous les identifiants sont immédiatement lisibles. Risque métier : compromission totale des comptes en quelques secondes.
🔴 CRITIQUE — Mot de passe admin codé en dur
DataLoader.java initialise le compte administrateur avec le mot de passe admin123 écrit directement dans le code source, commité dans Git. Tout développeur ayant accès au dépôt connaît ce mot de passe. Risque métier : accès non autorisé à la console d'administration, vol de données clients.
🔴 CRITIQUE — Mots de passe exposés dans toString()
AdminLogin.toString() et UserLogin.toString() incluent le mot de passe en clair dans leur représentation texte. Si un framework de logging (Spring AOP, intercepteur HTTP, débogage) loggue ces objets, les mots de passe apparaissent dans les fichiers de log. Risque métier : exposition des credentials dans les journaux système.
🟠 URGENT — H2 Console exposée publiquement
La console H2 (/h2-console) est activée et accessible sans authentification forte (route whitelistée dans Spring Security). CVE-2022-45868 affecte les versions antérieures d'H2 Console avec exécution de code à distance. Risque métier : accès complet à la base de données sans credentials.
🟠 URGENT — BCryptPasswordEncoder déclaré mais jamais câblé
Un bean BCryptPasswordEncoder est déclaré dans SecurityConfig mais n'est injecté ni dans AdminServices ni dans UserServices. L'encodeur existe mais n'est pas utilisé, créant une fausse impression de sécurité. Risque métier : effort de sécurisation déjà partiellement payé mais sans effet.
🟠 URGENT — Mutations via @GetMapping (9 routes)
Les opérations de mise à jour et suppression (admin, produit, utilisateur) utilisent des requêtes HTTP GET. N'importe quel lien ou image sur une page tierce peut déclencher involontairement ces opérations (attaque CSRF-like, même sans token). Risque métier : suppression accidentelle ou malveillante de données.
🟡 ATTENTION — 4 services sans @Transactional
Aucun service ne déclare @Transactional sur ses méthodes de mutation. En cas d'erreur partielle lors d'une sauvegarde multi-entités, la base de données peut se retrouver dans un état incohérent (données à moitié écrites). Risque métier : corruption silencieuse des commandes ou des profils clients.
🟡 ATTENTION — Couverture de tests 67 % (cible 80 %)
Le ratio fichiers de test / sources est de 67 %. Plusieurs comportements critiques (validation des credentials, logique de commande, gestion des sessions) ne sont pas entièrement couverts. Risque métier : régression non détectée lors des montées de version ou des refactorisations.
