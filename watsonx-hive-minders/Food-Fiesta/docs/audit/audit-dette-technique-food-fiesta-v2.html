2. Alertes Proactives

✅ RÉSOLUCouverture de tests — passée de 4% à 33%
63 tests unitaires et d'intégration ont été ajoutés (Mockito + @DataJpaTest H2), couvrant les 4 services, les 3 repositories et la logique de calcul. La dette tests a baissé de 3 jours, économisant 1 650 €. Prochaine cible : 60% de couverture sur les services critiques d'authentification.
🔴 CRITIQUEMots de passe stockés et comparés en clair — non corrigé
Les mots de passe clients et administrateurs sont toujours comparés en texte brut (UserServices:58, AdminServices:55). Le BCryptPasswordEncoder est déclaré mais non câblé. En cas de fuite de base de données, tous les comptes sont compromis instantanément. Risque RGPD, atteinte à la réputation, responsabilité légale.
🔴 CRITIQUEMot de passe admin "admin123" codé en dur — non corrigé
DataLoader.java:54 — le mot de passe administrateur est inscrit en clair dans le dépôt Git et loggé au démarrage. Toute personne ayant accès au code source ou à une image Docker peut accéder à l'interface d'administration. Prise de contrôle totale de l'application en quelques secondes.
🟠 URGENTConsole H2 exposée sans authentification — non corrigé
spring.h2.console.enabled=true + whitelist /h2-console/** dans SecurityConfig. Tout utilisateur non authentifié peut accéder à l'interface web complète de la base de données. Extraction de la totalité des données en quelques clics.
🟠 URGENT8 mutations exposées via @GetMapping — non corrigé
Les routes update/delete utilisent @GetMapping avec CSRF désactivé. Un lien malveillant dans un email peut déclencher une suppression sans aucune interaction consciente de l'administrateur. Sabotage des données via une simple URL.
🟡 ATTENTIONPlaceholders OAuth2 toujours dans le fichier de config versionné
YOUR_CLIENT_ID / YOUR_CLIENT_SECRET en lignes 17–18 de application.properties. Risque de commit accidentel de vraies clés OAuth en remplacement des placeholders. Utiliser des variables d'environnement ${GOOGLE_CLIENT_ID}.
🟡 ATTENTIONCouverture tests encore à 33% — cible 80% non atteinte
La progression est significative (+29 points) mais les contrôleurs, le flux OAuth2, et le DataLoader ne sont pas encore couverts. Continuer avec des tests d'intégration @WebMvcTest pour les contrôleurs critiques (AdminController, UserController).
